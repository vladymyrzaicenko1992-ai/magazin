# Project Selection Notes

Этот документ фиксирует, что именно выбрано как "топ" версия для отдельного GitHub проекта.

## Сводка по версиям

### `magaz55`
- Минимальный набор (`index.html`, `images`).
- Подходит только как черновая витрина.

### `magaz4`
- Есть базовые API/админ файлы и стартовые скрипты.
- Меньше целостности, чем в более новых версиях.

### `magaz 3.0`
- Сильные UI-идеи: сценарии, richer UX, расширенная клиентская логика.
- Часть функционала выглядит как экспериментальная надстройка.

### `magazinn`
- Набор API/админки и спецификационный txt-файл.
- Не выглядит как финальная сборка.

### `magazin2.0/magazinn`
- Самая подробная документация (`README.md`), есть Node/Express слой.
- Полезно как источник требований, чеклистов, API-подхода.

### `magaz-new` (выбрано как основа)
- Полный PHP-проект магазина с понятной структурой.
- Есть `SITE_MAP.md` и SQL-структура.
- Наиболее готов к упаковке и публикации.

## Финальное решение

- **Base for GitHub:** `magaz-new`
- **Reference docs:** `magazin2.0/magazinn/README.md`
- **Optional UX borrow:** отдельные идеи из `magaz 3.0`

## Критерии выбора

- Структурная целостность
- Простота деплоя
- Наличие SQL и API
- Минимум риска при переносе
