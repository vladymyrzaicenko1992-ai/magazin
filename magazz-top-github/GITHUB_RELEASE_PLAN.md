# GitHub Release Plan

План подготовки "топ" репозитория к публичному размещению.

## Этап 1. Сборка каркаса

- Создать новый репозиторий `magazz-top`.
- Перенести код из `magaz-new` в целевую структуру:
  - `public/` для веб-части
  - `app/` для серверных модулей (`api`, `admin`, `includes`)
  - `database/` для SQL
- Привести относительные пути после перемещения.

## Этап 2. Техническая чистка

- Удалить временные/локальные артефакты.
- Проверить конфиги БД (без паролей в репозитории).
- Убедиться, что пути к фото и ассетам корректны.
- Подготовить `.env.example` (если вводится env-конфигурация).

## Этап 3. Документация

- README:
  - описание проекта
  - стек
  - установка
  - запуск локально
  - деплой
- `docs/architecture.md` на основе `SITE_MAP.md`.
- `docs/api.md` с endpoint-ами.

## Этап 4. Проверка перед публикацией

- Открытие главной страницы.
- Работа каталога и категорий.
- Добавление в корзину.
- Отправка заказа в WhatsApp.
- Проверка API-методов.
- Импорт БД с нуля.

## Этап 5. Публикация

```bash
git init
git add .
git commit -m "Initial release: Magazz Top project"
git branch -M main
git remote add origin <your-repo-url>
git push -u origin main
```

## Минимальный Definition of Done

- Проект запускается локально без ручных правок в коде.
- README достаточен для стороннего разработчика.
- Нет секретов и мусора в репозитории.
- Основной пользовательский сценарий (поиск -> корзина -> заказ) работает.
