# Telegram-замовлення (mini CRM)

Сайт → кошик → **Google Apps Script** → бот **@Magazine1304_bot** → група «Заказы».

## 1. Бот уже создан

- Бот: [@Magazine1304_bot](https://t.me/Magazine1304_bot)
- Группа: [посилання на групу](https://t.me/+Oqqf9ywZPIs2ZDhi)

## 2. Добавить бота в группу

1. Откройте группу «Заказы».
2. Добавьте **@Magazine1304_bot**.
3. Сделайте бота **администратором** (право писать сообщения).

## 3. Узнать CHAT_ID группы

1. Напишите в группе любое сообщение (например `test`).
2. В браузере откройте (подставьте свой токен):

   `https://api.telegram.org/bot<TOKEN>/getUpdates`

3. Найдите `"chat":{"id":-100...` — это **TELEGRAM_CHAT_ID** (отрицательное число для группы).

Или после обновления Apps Script:

`https://script.google.com/.../exec?action=chats`

(нужны Script Properties с токеном).

## 4. Google Apps Script

1. Откройте таблицу → **Расширения → Apps Script**.
2. Вставьте код из `scripts/google-apps-script.gs` (замените старый).
3. **Проект → Настройки проекта → Свойства скрипта**:

   | Ключ | Значение |
   |------|----------|
   | `TELEGRAM_BOT_TOKEN` | токен от @BotFather |
   | `TELEGRAM_CHAT_ID` | id группы, напр. `-1001234567890` |

4. **Развернуть → Новое развертывание → Веб-приложение** (доступ: Все).
5. URL веб-приложения должен совпадать с `assets/data/config.json` → `googleWebAppUrl`.

## 5. Проверка

1. На сайте добавьте товар в корзину.
2. Укажите имя и телефон → **Оформити замовлення**.
3. В группе должно появиться сообщение **🛒 НОВЕ ЗАМОВЛЕННЯ**.

История заказов пишется на лист **orders** в той же Google Таблице.

## PHP (опционально)

Если появится сервер с PHP:

1. Скопируйте `api/config.example.php` → `api/config.php`.
2. Укажите токен и chat_id.
3. В `config.json` добавьте: `"orderApiUrl": "https://ваш-домен/api/order.php"`.

## Безопасность

- **Не коммитьте** токен бота и `config.php` в GitHub.
- Если токен попал в чат — перевыпустите в @BotFather → `/revoke`.
